反思一次失败的微软笔试

  • 笔试

博客荒废了一年多，其间发生了很多事，应该说是对我人生意义重大的一年。这一年中，确定了自己今后的发展方向，同时为这个方向做了很多准备，可是很快就遭遇了挫折——记一次失败的笔试。

首先介绍一下微软的这个预科生计划。
每年3月开放申请，4月初笔试，月底面试，5月发offer,6月入职。
由于预科生计划的主要目的在于选拔优秀的人才以备转正（转正概率较大），因此笔试的难度还是不小的（甚至超过）。总共四题，拿LeetCode作对比，四道题的难度大概分别在Medium，Medium，Medium，Hard级别，2个半小时内AC尽量多的题目，按排名进入面试。题目在这里 。

下面来说说自己，从2月底开始刷LeetCode，中间应付各种课程和作业，实力方面应该还欠缺许多，但是从往年的题目完成来看，AC两道应该没问题，属于刚好可以进入面试的程度，但是这次笔试缺遭遇滑铁卢。除去本次笔试难度上升的因素外，自身能力以外的许多问题在这一过程中暴露严重，简单总结，自勉以共勉。

  1. 对时间的控制
    在这次笔试之前没有进行过对时间控制的模拟练习，时间控制过于自由，造成笔试时时间控制糟糕，特别是在第一题卡住的时候，前前后后翻来覆去，直接心态爆炸。因此以后会加强时间控制以及调度的练习。
  2. 心态
    心态，心态，心态。这次所有的不甘心都来自于对心态的把控。从头到尾都没有真正的冷静下来，思路一团糟。
  3. 想好再写
    这次笔试很多人卡在了第一题，原因就是第一题带有一些误导性的内容，其实只是递归推导数学期望公式，但是题目中愣是把一棵二叉树放在上面。我看到题目后就急于动手，说来惭愧，前一个小时都用在了创建二叉树上面。。。后来想想其实根本不用建树（第二题同理），直接推导就可以了。想起来的时候时间已经过去了大半，心态就已经崩了。
  4. 应试技巧
    我只想吐槽线上OJ的形式让我想起了“一考定终身”的应试教育。（这里先不吐槽哪些比LeetCode差了十万八千里的各种OJ平台）
    用高考去定义一个人是不公平的，但是高考本身（监考、改卷相对严格）却是公平的。
    而用OJ笔试去定义一人的编程能力是公平的，但OJ本身却不公平，在不开摄像头，不限制切换次数的情况下，很多应试技巧甚至作弊都很容易发生。
    可接受的范围内，准备一些常用的模板会使coding效率上升，从Tree操作到各种图算法，甚至回溯、DP都总结出了框架，可以直接套用。

想说的就这么多，微软还是希望能进的，好好准备一年，明年再来吧。
